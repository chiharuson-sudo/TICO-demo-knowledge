# GitHub へのプッシュ手順

## ステップ0：Git をインストール（まだの場合）

1. **https://git-scm.com/download/win** を開く
2. 「Download for Windows」でインストーラーをダウンロード
3. 実行してインストール（基本的に「Next」でOK）
4. **PCを再起動**するか、**新しいターミナル**を開く

---

## ステップ1：GitHub でリポジトリを作る

1. **https://github.com** にログイン（アカウントがなければ「Sign up」で作成）
2. 右上の **「+」→「New repository」**
3. 次のように入力：
   - **Repository name**: 例）`tico-knowledge-graph-poc`
   - **Public** を選択
   - **「Add a README file」はチェックしない**（中身はローカルから上げるため）
4. **「Create repository」** をクリック
5. 次の画面で表示される **リポジトリのURL** をコピー  
   例：`https://github.com/あなたのユーザー名/tico-knowledge-graph-poc.git`

---

## ステップ2：プロジェクトフォルダで Git を初期化してプッシュ

**PowerShell または コマンドプロンプト** を開き、次を **1行ずつ** 実行してください。

```powershell
# プロジェクトフォルダに移動（パスは自分の環境に合わせてください）
cd "c:\internship\豊田自動織機\ナレッジグラフ用"

# Git の初期化（まだの場合）
git init

# すべてのファイルを「ステージ」に追加
git add .

# 最初のコミット
git commit -m "TICO ナレッジグラフ PoC 初回"

# GitHub のリポジトリを「リモート」として登録
# ※ YOUR_USERNAME と REPO_NAME を、ステップ1で作ったリポジトリのURLに合わせて書き換え
git remote add origin https://github.com/YOUR_USERNAME/REPO_NAME.git

# メインブランチの名前を main にする（初回だけ）
git branch -M main

# プッシュ（GitHub に送る）
git push -u origin main
```

### 例（ユーザー名が `tanaka`、リポジトリ名が `tico-knowledge-graph-poc` の場合）

```powershell
git remote add origin https://github.com/tanaka/tico-knowledge-graph-poc.git
```

---

## ステップ3：初回プッシュ時にログインを求められたら

- **ユーザー名**: GitHub のユーザー名
- **パスワード**: 通常のパスワードではなく、**Personal Access Token (PAT)** を使います。

### Personal Access Token の作り方

1. GitHub で **右上のアイコン → Settings**
2. 左メニュー一番下の **「Developer settings」**
3. **「Personal access tokens」→「Tokens (classic)》**
4. **「Generate new token (classic)》**
5. Note に「vercel-deploy」など好きな名前を入力
6. **Expiration** で有効期限を選ぶ（例：90 days）
7. **repo** にチェックを入れる
8. **「Generate token」** を押す
9. 表示された **トークン（ghp_ で始まる文字列）をコピー**（あとで再表示できません）
10. プッシュ時に「Password」を聞かれたら、このトークンを貼り付ける

## うまくいかないとき

| 症状 | 対処 |
|------|------|
| `git は認識されません` | Git をインストールしたあと、**新しいターミナル**を開き直す |
| `Permission denied` | GitHub にログインできていない。PAT を使っているか確認 |
| `remote origin already exists` | すでにリモートが登録済み。`git push -u origin main` だけ実行 |
| `failed to push` | `git pull origin main --rebase` を実行してから、もう一度 `git push` |
