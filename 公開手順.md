# TICO ナレッジグラフ PoC — みんなに公開する手順

## 方法1: Vercel で公開（おすすめ）

無料でURLが発行され、誰でもアクセスできます。

### 1. コードを GitHub に上げる

```bash
# まだの場合
git add .
git commit -m "ナレッジグラフPoC 公開用"
git branch -M main
git remote add origin https://github.com/あなたのユーザー名/リポジトリ名.git
git push -u origin main
```

### 2. Vercel にデプロイ

1. **https://vercel.com** にアクセス → アカウント作成（GitHubでログインが簡単）
2. **「Add New…」→「Project」** をクリック
3. **「Import Git Repository」** で GitHub のリポジトリを選択
4. 設定はそのままで **「Deploy」** をクリック
5. 数分後に **「https://〇〇.vercel.app」** のようなURLが発行されます

### 3. URLを共有

- 発行されたURLをメールやチャットで共有すれば、同じURLを開くだけで誰でもアプリを見られます。
- 必要なら **Vercel のダッシュボード → Settings → Domains** で独自ドメインも設定できます。

## 方法3: GitHub Pages で公開

無料で `https://あなたのユーザー名.github.io/リポジトリ名/` に公開できます。

1. **vite.config.ts** に `base: '/リポジトリ名/'` を追加する（リポジトリ名は実際の名前に合わせる）
2. `npm run build` で `dist` を生成
3. GitHub リポジトリの **Settings → Pages** で、Source を **GitHub Actions** にする
4. リポジトリに `.github/workflows/deploy.yml` を追加して、ビルド＆デプロイ用のワークフローを設定

（GitHub Pages を使う場合は、そのための設定ファイルも別途用意できます。）

---

## 注意事項

- **公開範囲**: このURLを知っている人は誰でも内容を見られます。社外秘のデータが含まれる場合は、リポジトリを private のままにして、Vercel の「Deploy」だけ有効にする方法もあります。
- **更新**: GitHub に `git push` すると、Vercel が自動で再デプロイし、公開URLの内容が最新になります。

まずは **方法1（Vercel + GitHub）** で進めるのが簡単です。
